红蓝对抗应急响应之windows

现场信息判断

首先判断被入侵的是什么区域，是否部署了阻断产品 。分成三个区域

  • 外网
  • DMZ
  • 内网

（在某些甲方的称呼中，此处的内网相当于该甲方的外网，此处的外网相当于该甲方的互联网，该甲方的内网实施物理隔离，禁止网络接口，属于涉密的机关办公业务网络，文件传输需要打申请，不允许使用U盘，一般使用光盘）

再判断目前是什么情况

是对抗被打了还是日常巡检被打了，还是被挖矿了

如果是员工机，一时间先断网，断网前加个快照，断网后加个快照，把数据脱下来慢慢看。在看看被入侵的机器数量，看看是因为什么原因沦陷的，是系统漏洞还是弱口令，还是邮件钓鱼，如果是邮件钓鱼，看看内网邮箱还是外网邮箱，邮件肯定是群发的，排查哪些员工点过邮件，着重排查点过邮件的员工电脑

如果是DMZ的客户资产，先救数据！！！

如果是资产，先检查账号安全

检查当前的登陆账户

qurey user

如果有异常用户，就先把异常用户踢出用户

如果是攻防的话，管理员上线红队就知道了，现在都会监控管理员是否上线的

或者检查lusrmgr.mcs

管理员组有没有新增的账户

很多人在渗透的时候会创建隐藏用户

隐藏账户的话一般用D盾，

或者真要在真实环境搞的话，在注册表管理，把当前用户暂时改成完全控制，然后检查本地账户有没有新增加的账户

再查看有没有异常的网络连接

检查下有没有异常代理 因为攻击者cs很可能会再搞个跳板什么的

网络配置设置那一块

再检查下有没有异常的端口，攻击者cs可能不会在一个地方放监听器

这个地方可能放了监听器

route print 检查下路由表

检查下一跳有没有异常

检查进程

这个地方用火绒剑 ，检查哪些系统进程，哪些是普通应用进程，没有证书的应用会被标注出未知应用

或者tasklist 然后可以用findstr 筛选，可以taskkill 杀进程

检查启动项

看看系统开机时间，有没有自启任务 ，看看有没有什么异常的启动信息

检查浏览器的访问记录

直接打开浏览器看看
