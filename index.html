基于powershell封装的docker工具

前因

之前从linux转到windows的时候就有很多地方不习惯，会有很多linux原生工具在windows会无法使用，或者出各种各样的问题，就比如rustscan （这个官方给出的解决措施是docker 加上powershell 别名来解决） 但是我在使用的过程中发现了有很多的问题

这个是docker pull docker镜像的命令 这个没问题

这个是docker 运行 rustscan的命令

很显然，在实际过程中 我们不可能跑这么长的命令

官方给出的解决办法

我按照官方的办法来使用

这部分出现了问题，报错是这样的

很显然，这个alias 是在linux 命令下的别名操作

官方的docker下载方法文档是基于linux的环境的

在windows 的powershell 虽然移植了这个命令，但是还是不一样

我们来看看alias的指令

这是微软官方是alias的指令文档

https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/add-alias

windows 的别名只能将一个比较长的别名（必须是一个单词转成另外一个比较短的单词）

途中我找了其他的绕过方法，有说{}将长命令内容包裹起来的

有说用“”或者‘’包裹长命令

我都试过了，没用

我甚至还想办法绕过空格，比如<>绕过 () 绕过,+绕过啥的，也失败了

所以 使用alias方法暂时没用

后果

后来从别人博客看到，他们通常使用powershell来包装长的docker语句

很多人还专门写了powershell脚本来管理docker集群

（这些在github上可以找到）

然后我就想，能否用powershell 的函数功能来实现这个功能

在powershell 中 $Profile 是个环境变量 ，

在powershell中输入这个会出现一个像这样的文件路径（注意，每个人的可能不一样，这里不要复制，这是我的环境的）

每次打开powershell 会加载这个文件

然后将这个文件复制进文件管理器

会用记事本打开一个文件

我在其中添加了这个

function rustscan {docker run -it --rm rustscan/rustscan:2.0.0 $args $args $args $args $args $args $args $args}

其中$args 是预留位，给之后可能会出现的参数作准备的 如果没有添加这个，会出现后面输入参数无效的情况

然后保存
